# V3.8.6 Production Security

## Turnstile

1. Create a Cloudflare Turnstile widget for the production domain and `localhost`.
2. Set `VITE_TURNSTILE_SITE_KEY` in Vercel.
3. Set `TURNSTILE_SECRET_KEY` and `TURNSTILE_ALLOWED_HOSTNAMES` in Railway.
4. In Supabase, open Authentication > Attack Protection, enable Cloudflare Turnstile, and add the same secret key.
5. Redeploy both services after changing environment variables.

The browser sends the one-time token directly to Supabase for email sign-in, registration, and password recovery. Google OAuth first validates its token through the public backend verification endpoint.

## DDoS Boundary

Turnstile blocks automated form abuse but is not a complete DDoS service. Put a custom domain behind the Cloudflare proxy, enable managed WAF rules, and add rate-limiting rules for:

- `/api/security/turnstile/verify`
- Supabase authentication endpoints
- Expensive analysis and history endpoints

The backend includes a small per-instance burst guard. Keep `API_RATE_LIMIT_PER_MINUTE=0` when Cloudflare handles general API rate limiting, or set a measured limit above the application's normal polling volume.
